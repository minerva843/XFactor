IMPORTANT

SVG Avatars Generator script uses an AJAX technology for the working and has a PHP server part. Thus it will not appear on your page if that page is opened directly from your HDD in a browser (like file:///C:/path/to/your_page.html). Use this script on a working local (like XAMP, LAMP, etc.) or a global web-server.

1. Overview of the pack

After downloading you will find the following folders in the archive:

  • documentation
  • for_develop
  • for_upload

documentation: As title said this is the documentation.

for_develop: This folder contains only one JavaScript file: svgavatars.core.js, which is a non-minified version of the main script for your customization.

for_upload: This one is for uploading to your server. The folder contains all the necessary files in a minified form to reduce your bandwidth and loading time.

2. Update (ver. 1.5)

Important: There is a new file structure since ver 1.4 and up. All the language files now are placed in the svgavatars > js > languages folder, and they are completely recoded. Also all the options are placed in a separate JS file: svgavatars > js > svgavatars.defaults.js

For correct work of this script please follow these steps:

  1. If you made any customization in CSS file svgavatars > css > svgavatars.css please backup it first.
  2. If you use "Save" function, backup all users created avatars from svgavatars > ready-avatars folder
  3. If you made any correction of existing translation or made your owns, backup them first.
  4. If you made any changes of settings, please backup them first.

After that please update the new files from ver 1.5 with your backed up data, delete all files from svgavatars folder and upload new ones with your new customization.

3. Setup

  1. Open the svgavatars.defaults.js file from the for_upload > LivIconsEvo > svgavatars > js folder in preferred text editor (not in MS Word!).
  2. Adjust the default options inside svgAvatarsOptions function, according to the information below. All the options has a comment line, however here are special notes for some of them:
    • Please pay attention to the pathToFolder option. If you upload the svgavatars directory to the root one on your server, the value of pathToFolder option should be either '/svgavatars/' or 'http://www.your_site.com/svgavatars/' or 'https://www.your_site.com/svgavatars/' if you use an encryption.
    • delta_sat and delta_val are used for calculating shadows and hightlights of choosen colors in the script itself. The HSV (hue, saturation, value) model is used for this process. Please look at this picture:



      The higher value of delta_sat and delta_val - the more contrast will be implemented. You can play with this settings, but please don't set very high values.
    • For correct work of uploading of created avatars to secure.gravatar.com you must have PHP cURL extension is enabled on your server (contact your hosting provider of how to). Also see PHP: cURL - Manual for more information.
  3. Upload to your server all the files and directories from the for_upload folder.
  4. Please be sure that directories' permissions of ready-avatars and temp-avatars are set to 755 (better choice) or 777 (not recommended).
  5. If you already use jQuery on your site do not upload it again.

4. Avatars files

All avatars files are split into two part: temporary and saved ones.

Temporary avatars files, which users download and share, are stored in the svgavatars > temp-avatars folder and can be periodically deleted with the help of cronjob, for example (please ask your hosting provider of how to).

All avatars saved by users on your server (with the help of hideSaveButton: false option) are stored in the svgavatars > ready-avatars folder. You can also put links to these files into your custom database, but you need to write your own additional PHP code, which you can add into the svgavatars > php > save-ready-avatar.php file.

5. HTML Structure

After uploading you need to declare a CSS file in the <head> section and JavaScript files before the closing </body> tag in your HTML page's code.

NOTE

It's recommended to place JS declarations before the closing </body> tag at the end of your HTML document, not inside the <head></head> section.

JavaScript declarations can be done in the two (2) ways:

EITHER

If you already use any components of tools JavaScript pack, include only the missing ones.

<head>
    ...
    <!-- all your other CSS files go here-->
    ...
    <link href="svgavatars/css/spectrum.css" rel="stylesheet">
    <link href="svgavatars/css/svgavatars.css" rel="stylesheet">
    <https://fonts.googleapis.com/css?family=Roboto+Condensed:400,300,700|Roboto:400,300,500,700&subset=latin,cyrillic-ext,cyrillic,latin-ext" rel="stylesheet">
</head>

<body>
    ...
    <!-- All your page's code goes here -->
    ...
    <script src="path/to/your_jQuery_copy.js"></script>
    ...
    <!-- Your other script files go here -->
    ...
    <script src="svgavatars/js/tools/svg.min.js"></script>
    <script src="svgavatars/js/tools/jquery.scrollbar.js"></script>
    <script src='svgavatars/js/tools/spectrum.min.js'></script>
    <script src="svgavatars/js/tools/rgbcolor.js"></script> 
    <script src="svgavatars/js/tools/StackBlur.js"></script>
    <script src="svgavatars/js/tools/canvg.min.js"></script>

    <script src="svgavatars/js/svgavatars.defaults.js"></script>
    <script src="svgavatars/js/languages/svgavatars.en.js"></script>
    <script src="svgavatars/js/svgavatars.core.min.js"></script>
</body>

Change svgavatars/js/languages/svgavatars.en.js file with your localized one.

The order of the declarations is important.

If you already use jQuery on your site do not declare it again. If not yet, please don't forget to edit the path/to/your_jQuery_copy.js

OR

You can declare only the one JS file LivIconsEvo.Tools.js, which contains all the necessary libraries.

<head>
    ...
    <!-- all your other CSS files go here-->
    ...
    <link href="svgavatars/css/spectrum.css" rel="stylesheet">
    <link href="svgavatars/css/svgavatars.css" rel="stylesheet">
    <https://fonts.googleapis.com/css?family=Roboto+Condensed:400,300,700|Roboto:400,300,500,700&subset=latin,cyrillic-ext,cyrillic,latin-ext" rel="stylesheet">
</head>

<body>
    ...
    <!-- All your page's code goes here -->
    ...
    <script src="path/to/your_jQuery_copy.js"></script>
    ...
    <!-- Your other script files go here -->
    ...
    <script src="svgavatars/js/svgavatars.tools.js"></script>
    <script src="svgavatars/js/svgavatars.defaults.js"></script>
    <script src="svgavatars/js/languages/svgavatars.en.js"></script>
    <script src="svgavatars/js/svgavatars.core.min.js"></script>
</body>

Change svgavatars/js/languages/svgavatars.en.js file with your localized one.

The order of the declarations is important.

If you already use jQuery on your site do not declare it again. If not yet, please don't forget to edit the path/to/your_jQuery_copy.js

Now you are ready to use SVG Avatars Generator.

6. CSS Structure

All the necessary styles are placed in the svgavatars > css > svgavatars.css file.

If your site design let you to expand the generator to full width you can delete "max-width" value at the top of the CSS file or set your own one:

#svgAvatars {
  max-width: 960px;
  margin: 0 auto;
}

If you will not use "Roboto" font, change line 39:

font-family: Roboto, "Helvetica Neue", Helvetica, Arial, sans-serif;

If you want to change any color scheme please find this section near the end of the CSS file:

/******************* Color schemes *******************/

/* LIGHT scheme */
#svga-container.svga-light,
...

7. Integration

And now all you need is to place the one line of code somewhere in your <body> section

<div id="svgAvatars"></div>

The id value must be exactly svgAvatars (case sensitive).
